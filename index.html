Featured · Published

RAG-Based Video Question Answering with Timestamp Localization

Problem Searching inside a video is painful — you know the answer is in there somewhere, but not where.

Approach Transcribed video into text, chunked and embedded the transcript, and stored it in a vector index. When a question comes in, the system retrieves the most relevant chunks and grounds an LLM's answer (served through Ollama) in that retrieved text — instead of letting the model guess. Each retrieved chunk carries its original timestamp, so the answer links straight back to the exact moment in the video.

Result Paper accepted for presentation at INCECT 2026.

AI-Powered HR Grievance Management System

Problem Employee grievances get triaged manually, which is slow and inconsistent across HR teams.

Approach Built an NLP classifier to route incoming grievances by category and urgency, then layered in RAG so the system pulls relevant policy context when generating a response — rather than answering from scratch. Wrapped it in a submission-and-status-tracking interface so employees can follow a grievance end to end.

Result A working prototype that replaces manual sorting with classify → retrieve → respond.

Earthquake-Stabilized Dialysis System

Problem A dialysis machine mid-treatment during an earthquake is a real safety risk — someone needs to detect that risk fast enough to act.

Approach Built the ML module for Team ResQ: fed real-time accelerometer/vibration data through a time-series model trained to recognize earthquake-like patterns, and used that prediction to trigger AI-assisted safety decisions — like pausing or stabilizing the machine — before a human could intervene manually.

Result Selected in KLE Technological University's internal Smart India Hackathon 2025 round.
